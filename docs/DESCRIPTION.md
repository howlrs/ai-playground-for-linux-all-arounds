# 説明書 - AI Playground for Linux

## このプロジェクトは何か

AIコーディングエージェントが自由に動作できるLinux開発環境を、Dockerコンテナとして
提供するものです。`docker compose up` 一発で、AIエージェント・プログラミング言語・
クラウドCLI・業務ツールが揃った環境が立ち上がります。

## 設計思想

### 疑問と探求

- AIエージェントがコードを書き、自らテストし、疑問があれば別のAIにQAを依頼する
- Claude Code（主）がGemini CLI（従）にレビューを投げる運用を想定
- 人間は方針を示し、AIが探求して実装する

### 真の貢献

- 環境構築に時間を取られず、本質的な開発作業に集中できる
- 再現性のあるDockerfile/compose.yamlにより、どの環境でも同じ開発体験を得られる

## なぜ単一コンテナか

Claude CodeがGemini CLIをサブプロセスとしてQAに使う運用では、同一ファイルシステム上で
動作する必要がある。コンテナ間通信のオーバーヘッドを避け、シンプルな構成を維持するため、
単一コンテナに全ツールを搭載している。

## Python環境: なぜuvか

| 項目 | uv | conda | pip+venv |
|------|-----|-------|----------|
| インストール速度 | 極めて高速 | 遅い | 普通 |
| イメージサイズ影響 | 小 (+~50MB) | 大 (+1GB以上) | 中 |
| venv互換性 | 完全互換 | 独自形式 | 標準 |
| Pythonバージョン管理 | `uv python install` | conda自体で管理 | pyenv等が別途必要 |
| ロックファイル | `uv.lock` | `environment.yml` | なし（要pip-tools） |

uvはRust製で高速、pip/venvと完全互換、Pythonバージョン管理も内蔵。
Dockerイメージの肥大化を防ぎつつ、モダンなPython開発体験を提供する。

## 権限モデル

コンテナ内ユーザー`dev`には`sudo NOPASSWD`を付与し、AIエージェントが
パッケージインストールやシステム設定変更を自律的に行えるようにしている。

**ただし**、外部サービスCLI（gh, glab, gcloud, aws, m365）については
操作種別に応じた確認ポリシーを設けている。詳細は `docs/CAUTION.md` を参照。

## 永続化

コンテナを`docker compose down`しても、以下は保持される：

- AIエージェントの設定・認証情報（Dockerボリューム）
- AgentDB のデータ（logs + knowledge）
- uvのパッケージキャッシュ（再インストール高速化）
- M365 CLIの認証セッション
- `./workspace`ディレクトリのファイル（バインドマウント）

`docker compose build` によるイメージ再構築でも上記は全て保持される。
`docker compose down -v` でボリュームも含めて完全リセット可能（**全データ消失に注意**）。

**重要:** 作業ファイルは必ず `~/workspace/` 内に保存すること。
`~/` 直下に置いたファイルはイメージ再構築時に消失する。
